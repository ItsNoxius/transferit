transfer.it → browser download

Paste a share URL. MEGA’s CDN only serves ciphertext, so this page can’t hand you a real https://…/file.zip link. Instead it queues a job and opens a same-origin URL that a Service Worker streams through AES-CTR decrypt into Chrome/Firefox’s normal download shelf — disk-backed, not held in a Blob.

    Serve over http://localhost (not file://): npx --yes serve examples. Needs a Service Worker (HTTPS or localhost). The MEGA CDN URL alone is never a usable direct download — opening it saves encrypted garbage. The only way to get a true https://yoursite/file.zip link is a server that decrypts and streams plaintext with Content-Disposition: attachment.
